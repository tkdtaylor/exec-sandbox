# Task 013: Firecracker tier dispatch + config-generator skeleton

**Status:** ⬜ backlog
**Branch:** `task/013-firecracker-tier-dispatch-config-skeleton`
**Spec:** [`docs/tasks/test-specs/013-firecracker-tier-dispatch-config-skeleton-test-spec.md`](../test-specs/013-firecracker-tier-dispatch-config-skeleton-test-spec.md)
**ADR:** ADR 010 (Firecracker Tier-3 backend) — accepted. This task implements D1 (backend behind `backendFor`), D2 (no-NIC by omission, at the config level), D3 (config as a pure function of on-host paths). No new ADR required.

## Readiness

**READY.** This is the **thin first slice** of the Firecracker epic. It has **no open-question
dependency** — Q1 (kernel/rootfs sourcing), Q2 (mount mechanism), Q3 (jailer privilege), and Q4
(shim location) are all about *launching* and *wiring* the guest, none of which this task does. This
slice is **config generation + tier dispatch only**: a pure function unit-testable without
`/dev/kvm` or the `firecracker`/`jailer` binaries.

**Dependency position in the epic:** 013 → 014 → 015 → {016, 017} → 018. This is the root; nothing
in the epic precedes it.

## Problem

`backendFor(tier)` (`run.go:258-267`) wires `""`/`bubblewrap` → `bubblewrapBackend` and `gvisor` →
`gvisorBackend`; **every other tier, including `firecracker`, hits the default arm and returns
`tier not implemented: <tier>`**. SPEC.md Non-goals records `firecracker` as "accepted by the
`tier` field but returns `tier not implemented`." There is no Firecracker backend and no microVM
config generator.

ADR 010 (D1/D3) decided Tier-3 is Firecracker behind the seam, generated by a pure config builder
mirroring `gvisorOCISpec` (`gvisor.go:190` — a `map[string]any` built purely from on-host paths,
unit-testable without `runsc`). The load-bearing security property (D2) is that the generated config
carries **no `network-interface` key** — the microVM analogue of `bwrap --unshare-all`, "no network
by omission."

## Scope

- **Add `firecracker.go`** (mirroring `gvisor.go`) with:
  - `firecrackerBackend` satisfying the `Backend` interface (`run.go:251-253`). In this slice its
    `Argv` builds the config and writes it to a per-run bundle dir but **does not launch the VMM**
    (launch is task 015); it may return a not-yet-launchable argv or a clearly-marked stub for the
    spawn step — the **load-bearing deliverable is the config generator**, not a running guest.
  - `firecrackerConfig(kernelPath, rootfsPath, scriptPath, vsockUDS string, lim Limits)` — a **pure
    function** returning the microVM config (machine-config, boot-source, root-drive, vsock) built
    only from the supplied on-host paths. It issues **no `network-interface` body** (D2).
  - A reusable **no-NIC assertion helper** (e.g. `configHasNoNIC(cfg) error`) that task 014 and the
    task-018 `fitness-no-nic` rule reuse. It scans the config (structured + serialized) for any
    `network-interface`/`network-interfaces` key and errors if present.
- **Wire `backendFor`** (`run.go`): add a `case "firecracker": return firecrackerBackend{}, nil`
  arm. The default arm still rejects genuinely-unknown tiers (`qemu` → `tier not implemented: qemu`).
- **Determinism:** `firecrackerConfig` must serialize byte-for-byte identically for identical inputs
  (sorted keys / fixed field order — mirror `envSetenvPairs`/`gvisorOCISpec` reproducibility).
- **Spec update in the same commit (this is the implementation task):** rewrite the SPEC.md
  Non-goals "Tier 3 not yet implemented … returns `tier not implemented`" line in place to state
  that `firecracker` now dispatches to a config-generating Tier-3 backend, with the one-shot VMM
  launch as the current boundary (present tense — "the backend generates the microVM config; the
  VMM launch is not yet wired"). No future tense.

Out of scope: launching the VMM / driving the REST API (task 015); wiring the vsock to the live
proxy + the guest shim (task 014); the limits→machine-config mapping beyond what the config
*structure* needs (task 016 owns the mapping semantics); `/work`/FileRead presentation (task 017);
teardown + the `make fitness` umbrella wiring (task 018). Do NOT modify `gvisor.go`.

## Definition of done

- `firecracker.go` exists with `firecrackerBackend`, `firecrackerConfig` (pure), and the reusable
  no-NIC helper.
- `backendFor("firecracker")` returns a `firecrackerBackend`; `""`/`bubblewrap`/`gvisor` dispatch is
  unchanged; an unknown tier still errors.
- `firecrackerConfig` builds with **no** `/dev/kvm` / firecracker binary, carries **no**
  `network-interface` key (base + every shape this slice produces), wires the vsock `uds_path` /
  root drive / boot source / `sh /payload.sh` entry, and is byte-for-byte deterministic.
- The no-NIC helper passes on the real config and **fails** on a constructed NIC config (not a no-op).
- SPEC.md Non-goals rewritten in place (firecracker dispatches; VMM launch is the boundary); no
  future tense.
- `go test -count=1 ./...` green; `gofmt -l .` clean; `gvisor.go` unmodified.
